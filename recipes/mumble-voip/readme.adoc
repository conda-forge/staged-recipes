== System Library Integration

This recipe uses system-installed libraries from conda-forge instead of bundled third-party dependencies, providing:

* Reduced build times — no compilation of external libraries
* Smaller packages — shared libraries reduce disk usage
* Automatic security updates via conda-forge dependency management
* Better ecosystem integration with other conda packages

=== Libraries Still Bundled

These remain bundled because they are Mumble-specific forks with no upstream conda-forge equivalent:

* *ReNameNoise* — Mumble's noise suppression fork
* *mach_override* — macOS function overriding (Mumble-specific fork)

Build utilities bundled for the build process:

* *FindPythonInterpreter* — CMake module
* *cmake-compiler-flags* — CMake utilities
* *flag-icons* — UI assets

=== CMake Configuration

Two files are included alongside the recipe to integrate with conda-forge:

* `cmake_system_libs.cmake` — detects and configures system libraries via `find_package()` and `pkg-config`
* `conda_toolchain.cmake` — sets `CMAKE_PREFIX_PATH`, RPATH, and compiler flags for the conda environment

All bundled libraries are disabled at configure time via `-Dbundled-<lib>=OFF` flags.
See https://github.com/mumble-voip/mumble/blob/master/docs/dev/build-instructions/cmake_options.md#bundled-gsl for the full list of upstream CMake options.

== Recipe Files

=== `service.yaml`

Service descriptor for `mumble-server` on Linux. Consumed by a service-management tool at deploy time to generate the systemd unit and lay down runtime files. It specifies:

* The executable to run (`mumble-server -ini /etc/mumble/mumble-server.ini`) and its working directory (`/var/lib/mumble`)
* Runtime security: runs as `mumble:mumble`, systemd hardening (`NoNewPrivileges`, `ProtectSystem=strict`, `MemoryDenyWriteExecute`, private-IP network allowlist, etc.)
* Restart policy: on-failure, up to 3 restarts in 60 s with a 5 s delay
* File layout to provision on install: `/var/lib/mumble`, `/var/log/mumble`, `/etc/mumble`, a default `mumble-server.ini` (port 64738, SQLite database, 100-user limit), and `/etc/default/mumble-server`

Bundled into the package via:
[source,yaml]
----
- path: service.yaml
  target_directory: service-config
----
and installed to `${PREFIX}/etc/mumble/service.yaml` by `build-server.sh`.

=== `test_local_build.nu`

Developer convenience script for iterating locally before running the full rattler-build CI pipeline. Run it from the recipe directory:

[source,bash]
----
nu test_local_build.nu
----

It checks that `cmake`, `ninja`, and `git` are available, validates that `recipe.yaml` references the expected system dependencies, and runs a quick mock CMake configuration using `cmake_system_libs.cmake` and `conda_toolchain.cmake` against a minimal dummy `CMakeLists.txt`. It does *not* compile Mumble itself.

=== `tests/test_mumble_client.nu`

Post-install smoke test run by rattler-build for the `mumble-client` output. Verifies:

* `${PREFIX}/bin/mumble` (or `mumble.exe` on Windows) exists
* `--version` can be invoked without error (failure is tolerated in headless environments)
* Optional library paths under `${PREFIX}/lib/mumble/` are reported
* `${PREFIX}/share/licenses/mumble-client/LICENSE` is present (hard failure if missing)

=== `tests/test_mumble_server.nu`

Post-install smoke test run by rattler-build for the `mumble-server` output. Verifies:

* `${PREFIX}/bin/mumble-server` exists
* `--version` and `--help` can be invoked
* `${PREFIX}/etc/mumble/service.yaml` is present and parses as valid YAML
* `${PREFIX}/share/licenses/mumble-server/LICENSE` is present (hard failure if missing)
* On Linux: `ldd` output contains no "not found" entries (runtime library check)
